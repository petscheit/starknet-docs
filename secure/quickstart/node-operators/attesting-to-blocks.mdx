Starting from its second phase, the staking protocol requires validators to prove they are actively preserving the history of the network by submitting attestations to randomly assigned blocks in each epoch.

This installment of the series will therefore walk you through [running Equilibrium's attestation service](#running-pathfinders-attestation-service).

<Note>
To run a Nethermind's attestation service, see [the instructions in its GitHub repository](https://github.com/NethermindEth/starknet-staking-v2/tree/main?tab=readme-ov-file#starknet-staking-v2).
</Note>

## Running Equilibrium's attestation service

Luckily, running [Equilibrium's attestation service](https://github.com/eqlabs/starknet-validator-attestation/tree/main) is as simple as running:

```shell
docker run -it --rm --network host \
  -e VALIDATOR_ATTESTATION_OPERATIONAL_PRIVATE_KEY=$OPERATIONAL_PRIVATE_KEY \
  ghcr.io/eqlabs/starknet-validator-attestation \
  --staking-contract-address 0x03745ab04a431fc02871a139be6b93d9260b0ff3e779ad9c8b377183b23109f1 \
  --attestation-contract-address 0x03f32e152b9637c31bfcf73e434f78591067a01ba070505ff6ee195642c9acfb \
  --staker-operational-address $OPERATIONAL_ADDRESS \
  --node-url http://localhost:9545/rpc/v0_8 \
  --local-signer

```
